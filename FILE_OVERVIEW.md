## 🎬 推薦的 4 個演示 Prompts

### Prompt 1：基礎輸入驗證 ⭐
**難度：** 簡單  
**時間：** 2-3 分鐘  
**展示：** 前後端一致性修改

```
為這個用戶註冊系統添加完整的輸入驗證功能：
1. 前端驗證（signup.ts）：用戶名（3-20字符）、Email格式、密碼（至少8字符）
2. 後端驗證（service.go）：相同邏輯 + 錯誤訊息
3. 密碼強度驗證：大小寫字母 + 數字
```

### Prompt 2：密碼強度指示器 ⭐⭐
**難度：** 中等  
**時間：** 2-3 分鐘  
**展示：** UI 增強和多輪對話

```
添加視覺化的密碼強度指示器：
1. HTML 進度條元素
2. TypeScript 強度計算邏輯
3. 顏色：弱(紅)、中(黃)、強(綠)
4. CSS 樣式美化
```

### Prompt 3：錯誤提示優化 ⭐⭐
**難度：** 中等  
**時間：** 2-3 分鐘  
**展示：** UX 改進

```
優化錯誤處理和用戶體驗：
1. 即時表單驗證
2. 每個欄位的錯誤提示
3. 錯誤時紅色邊框
4. 成功時綠色勾選
5. 友好的中文訊息
```

### Prompt 4：添加快取（進階）⭐⭐⭐
**難度：** 進階  
**時間：** 3-5 分鐘  
**展示：** 智能提問和複雜任務

```
我想為這個應用添加快取功能來提升性能。

[Claude 會詢問細節，然後回答：]
- 快取層級：API 回應快取
- 目標：減少重複查詢
- 技術：內存快取
- 目標區域：用戶列表查詢
- TTL：5 分鐘
```
