## 🎯 演示流程（總時長：5-10 分鐘）

### 第一步：介紹專案背景 (30 秒)

**說明：**
> "這是一個簡單的用戶註冊系統，包含 Go 後端和 TypeScript 前端。你可以看到代碼中有很多 TODO 標記，這些是我們刻意留下的待完成功能，接下來我會用 Claude Code Web 來完成它們。"

**展示：**
- 打開 GitHub repository 頁面
- 快速瀏覽 `service.go` 和 `signup.ts`
- 指出 TODO 註釋

---

### 第二步：啟動 Claude Code Web (1 分鐘)

**操作步驟：**

1. **訪問** https://claude.ai/code

2. **連接 Repository**
   - 點擊 "Select a repo" 或 "+ New session"
   - 選擇 `lewis100challenges/claude-code-demo`
   - 確認選擇

3. **確認環境**
   - Claude 會自動建立隔離的沙箱環境
   - 顯示 repository 已連接

**說明：**
> "Claude Code Web 現在運行在雲端，我不需要開啟任何 IDE 或終端機，所有操作都在瀏覽器中完成。"

---

### 第三步：演示核心功能 - 添加輸入驗證 (3-4 分鐘)

#### 📝 Prompt 1: 基礎輸入驗證

**在 Claude Code Web 中輸入：**

```
為這個用戶註冊系統添加完整的輸入驗證功能：

1. 前端驗證（signup.ts）：
   - 用戶名：3-20 個字符，只能包含字母、數字和下劃線
   - Email：必須是有效的 email 格式
   - 密碼：至少 8 個字符

2. 後端驗證（service.go）：
   - 添加相同的驗證邏輯
   - 確保所有必填字段都存在
   - 返回清晰的錯誤訊息

3. 同時在前端和後端都添加密碼強度驗證：
   - 必須包含至少一個大寫字母
   - 必須包含至少一個小寫字母
   - 必須包含至少一個數字
```

**按下 Enter 並觀察：**

✨ **展示要點：**
- Claude 開始分析代碼
- 顯示正在修改的文件
- 實時看到代碼變更
- Claude 提供變更摘要

**說明：**
> "看，Claude 正在它的隔離沙箱中工作，我可以實時看到進度。它不僅修改了前端的 TypeScript 文件，還更新了後端的 Go 代碼，確保前後端驗證邏輯一致。"

---

### 第四步：展示 "Open in CLI" 功能 (2 分鐘)

**等 Claude 完成任務後：**

1. **查看變更摘要**
   - 閱讀 Claude 提供的修改說明
   - 查看改動的文件列表

2. **點擊 "Open in CLI" 按鈕**
   - 複製提供的命令

3. **在終端機中執行**
   ```bash
   # 貼上 Claude 提供的命令
   claude code open <session-id>
   ```

4. **展示本地環境**
   - 所有修改都已同步到本地
   - 可以在 VS Code 中查看變更
   - 可以看到完整的 git diff

**說明：**
> "這是我最喜歡的功能！只需一個按鈕，我就能把雲端的工作環境無縫轉移到本地。所有的上下文、修改歷史都完整保留。現在我可以在 VS Code 中繼續工作，或者運行測試。"

---

### 第五步：（可選）演示進階功能 - 多輪對話 (2-3 分鐘)

#### 📝 Prompt 2: 添加密碼強度指示器

**繼續在同一個 session 中輸入：**

```
很好！現在請添加一個視覺化的密碼強度指示器：

1. 在 HTML 中添加一個進度條元素
2. 在 TypeScript 中實現密碼強度計算邏輯
3. 根據密碼強度顯示不同顏色：
   - 弱：紅色
   - 中等：黃色
   - 強：綠色
4. 添加相應的 CSS 樣式
```

**展示要點：**
- Claude 記住之前的所有修改
- 在現有代碼基礎上繼續改進
- 保持代碼風格一致性

---

### 第六步：（進階）演示複雜任務分解 (選配，3-5 分鐘)

#### 📝 Prompt 3: 添加快取系統

**輸入一個較模糊的 prompt：**

```
我想為這個應用添加快取功能來提升性能。
```

**展示要點：**
- Claude 會主動詢問細節問題：
  - 快取層級？（資料庫查詢快取 vs API 回應快取）
  - 目標？（提升性能 vs 減少資料庫負載）
  - 技術選擇？（Redis vs 內存 vs 其他）
  
- **選擇回答：**
  ```
  - 快取層級：API 回應快取
  - 目標：減少重複查詢
  - 技術：內存快取（in-memory）
  - 目標區域：用戶列表查詢
  ```

- Claude 會根據你的回答實現相應功能

**說明：**
> "注意到了嗎？當我的需求不夠明確時，Claude 不會盲目執行，而是會主動提問，確保正確理解需求。這種交互式的工作方式就像在和一個真正的開發者合作。"

## 🎯 演示後的問答準備

### Q: 這和 GitHub Copilot 有什麼區別？
**A:** Copilot 主要是代碼補全工具，在你寫代碼時提供建議。Claude Code Web 則是一個完整的代理系統，它能自主分析、規劃、修改整個代碼庫，就像一個真正的開發者在工作。

### Q: 安全性如何？我的代碼會被存儲嗎?
**A:** Claude 在隔離的沙箱環境中運行，每個 session 都是獨立的。具體的數據政策建議查看 Anthropic 的官方文檔。

### Q: 可以處理多大的 repository？
**A:** Claude Code Web 可以處理相當大的代碼庫。對於演示，我們使用的是一個簡化的專案，但它能處理生產級別的應用。

### Q: 支援哪些程式語言？
**A:** Claude 支援所有主流程式語言，包括 Python、JavaScript、TypeScript、Go、Java、Rust 等等。

### Q: 要錢嗎？
**A:** Claude Code Web 目前對 Claude Pro 和 Team 用戶開放。具體定價請參考 claude.ai/code。
